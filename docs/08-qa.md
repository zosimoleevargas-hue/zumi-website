# Quality Assurance

## Objetivo

Garantizar que el sitio web de Zumi cumpla estándares profesionales de diseño, desarrollo, rendimiento, SEO y experiencia de usuario.

---

# Visual QA

## Diseño

Verificar:

- La interfaz coincide con el Design System.
- Los colores son consistentes.
- La tipografía mantiene jerarquía correcta.
- Los espacios entre elementos son adecuados.
- Los componentes tienen el mismo estilo.

---

# Responsive QA

Revisar en:

- Mobile.
- Tablet.
- Desktop.

Verificar:

- No existen elementos cortados.
- Los textos son legibles.
- Los botones son fáciles de tocar.
- Las imágenes mantienen proporción correcta.

---

# UX QA

Preguntas:

## Primera impresión

¿El usuario entiende qué es Zumi en menos de 5 segundos?

---

## Conversión

¿Es fácil pedir comida?

¿El botón de WhatsApp es visible?

---

## Navegación

¿El usuario encuentra rápidamente?

- Menú.
- Ubicación.
- Horarios.
- Contacto.

---

# Performance QA

Objetivos:

- Carga rápida.
- Imágenes optimizadas.
- Código eficiente.
- Evitar JavaScript innecesario.

Revisar:

- Lighthouse Performance.
- Core Web Vitals.

---

# SEO QA

Verificar:

- Metadata correcta.
- Title correcto.
- Description correcta.
- Sitemap funcionando.
- Robots configurado.
- Structured Data implementado.

---

# Accessibility QA

Verificar:

- Contraste correcto.
- Imágenes con alt text.
- Navegación con teclado.
- HTML semántico.
- Tamaño adecuado de botones.

---

# Code Quality QA

Verificar:

- Componentes reutilizables.
- Código limpio.
- Sin código duplicado.
- TypeScript correctamente utilizado.
- Sin errores en consola.

---

# Cliente QA

Antes de entregar:

Confirmar:

- Logo correcto.
- Fotografías aprobadas.
- Menú actualizado.
- Horarios correctos.
- Información de contacto correcta.
- Botones funcionando.

---

# Criterio de finalización

El proyecto se considera terminado cuando:

- Cumple la especificación.
- Pasa revisión visual.
- Tiene buen rendimiento.
- Está optimizado para Google.
- El cliente aprueba el resultado.
